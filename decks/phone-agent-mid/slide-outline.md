## Slide 2 — 문제 정의
- **누구의 문제?** 한국의 모바일 앱(배민, 카카오T, 캐치테이블 등)을 매일 쓰는 사람들
- **어떤 문제?** OpenClaw 같은 글로벌 AI 에이전트는 한국 로컬 앱에 접근 불가. 매번 직접 앱을 열고 수작업으로 주문/예약/호출해야 함.
- **빈도:** 매일 (택시, 배달, 예약 등)
- **고통 사례:** 팀끼리 배달음식 시킬 때 — 메뉴 수합하고, 앱 열어서 하나하나 장바구니에 담고 주문. 단순 반복인데 매번 손이 감.

## Slide 3 — 솔루션
- **한 문장:** "딸깍으로 한국의 모든 서비스, 에이전트에게 맡겨보세요"
- **핵심 흐름:**
  - 사용자가 자연어로 말한다 → AI가 의도를 파악 → 해당 앱을 자동 실행 → 실제 거래 완결
- **아키텍처:** User → Intent Parser (Claude) → Router → ServiceHandler → Android 자동화 (Droidrun)
- **차별점:** 브라우저가 아닌 **실제 모바일 앱**을 직접 제어 — 한국 서비스 대부분이 모바일 전용

## Slide 4 — 데모 시나리오
- 3개 시나리오 영상 시연:
  1. 🍽️ **캐치테이블** — 식당 예약 → 취소

## Slide 5 — 나의 랄프 세팅
- **AI 에이전트/모델:** Oh-my-openagent + Claude Opus 4.6
- **개발 방식:** AI와 페어 프로그래밍
- **핵심 스택:** Python + Droidrun SDK + Android Emulator + ADB
- **지속 전략:** AI가 코드를 짜고, 사람이 검증하고 방향을 잡는 루프
